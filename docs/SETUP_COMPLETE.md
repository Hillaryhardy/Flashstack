All documentation follows these standards:

### Structure
- Clear title and description
- Table of contents (for 50+ lines)
- Code examples with explanations
- Links to related docs
- Last updated date

### Naming
- Use descriptive names: `fee-mechanism.md`
- Include versions: `deployment-guide-v1.1.md`
- Use CAPS for important: `README.md`, `CHANGELOG.md`

- Review quarterly
- Archive outdated content
- Update links
- Keep examples working
